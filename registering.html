  • s3_base_path: S3 path for data storage
  • evaluator: Evaluator instance
  • model: Model instance

How the Model Registration Works

The model registration process follows these steps:

  1. Retrieving Evaluation Metrics:
    • The evaluation metrics file is retrieved from S3.
  2. Registering the Model:
    • The trained model is registered in the SageMaker Model Registry.
    • It is assigned an approval status (e.g., PendingManualApproval).
  3. Defining Model Parameters:
    • Supported content types, response types, and instance types are set.
  4. Storing the Model:
    • The model package is stored in the SageMaker Model Registry for later deployment.

Model Registry Configuration

Default configuration:

  • Content Types: text/csv
  • Response Types: text/csv
  • Inference Instances: ml.t2.medium
  • Transform Instances: ml.m5.large
  • Default Approval Status: PendingManualApproval
